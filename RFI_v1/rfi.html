Descripción de RFI / RFI Description

En base a la revisión de la biblioteca de antecedentes del proyecto (RCA, Ingeniería Básica VLP-470 y anexos), se solicita complementar y consolidar información para la correcta definición habilitante del expediente IFC y su correspondiente Permiso de Edificación:

1. Consolidación de Edificaciones del proyecto
Se identifican antecedentes parciales en arquitectura (5000), layout (4350) y RCA, sin consolidación única.
  1. Emitir listado oficial indicando tags por edificios/recintos) del proyecto (Administrativo, Operaciones, Equipamiento, Mantenimiento, Bodegas)
  2. Relacionar edificios con planos estructurales existentes (5230, 5240, 5250, 5260)
  3. Indicar si existen edificios adicionales no modelados en arquitectura
  4. Vincular cada edificio con su uso operacional
2. Planimetría consolidada del proyecto
Existen múltiples fuentes: layout general (4350), topografía (13140), KMZ y modelos 3D, sin integración única.
  1. Plano general consolidado georreferenciado (DWG base)
  2. Integración de layout proceso (4300) + arquitectura (5000)
  3. Definición de límites de edificaciones y áreas
  4. Coherencia entre KMZ, topografía y layout
3. Definición del Área de Proceso
Se dispone de P&ID (4340), diagramas de bloques (4310) y memorias (4130), pero no existe definición constructiva.
  1. Definir y aclarar qué área de proceso corresponde a estructura techada, abierta o mixta
  2. Delimitar áreas como losas, edificaciones, áreas de almacenamiento, etc.
  3. Indicar requerimientos estructurales (fundaciones, racks, soportes)
4. Infraestructura de almacenamiento (Tank Farm)
Se cuenta con memorias de cálculo (6210) y P&ID (4340), pero falta definición arquitectónica/constructiva.
  1. Plano de implantación de estanques (coherente con 4340 y 4350)
  2. Definición de pretiles (dimensiones y capacidad)
  3. Clasificación de áreas peligrosas (4160) aplicada a layout <---REVISAR SI APLICA para esta área---
  4. Relación con DS 43/2015 <---REVISAR SI APLICA para esta área---
5. Edificios de soporte y utilidades
Se identifican referencias en mecánica (6412 sala de utilities) y procesos (Nodo 4 Utilities), sin consolidación arquitectónica.
  1. Definir sala de calderas, compresores, oxidación térmica
  2. Plano y memoria de sala de subestación eléctrica
  3. Definir ubicación y características de generador diésel
  4. Relación entre utilities y área de proceso
6. Memorias técnicas por disciplina
Existen memorias separadas (eléctrica, hidráulica, proceso, mecánica), pero no integradas para IFC.
  1. Memoria descriptiva consolidada del proyecto (arquitectura + proceso)
  2. Memoria sanitaria (agua potable, servidas, RILES – ref. 4136)
  3. Memoria eléctrica (10210, 10220)
  4. Memoria de riesgos y áreas peligrosas (4160, HAZOP 4510)
7. Clasificación normativa y cumplimiento
Se requiere aterrizar normativa a nivel de edificación (no solo proceso).
  1. Clasificación de uso por edificio (OGUC)
  2. Carga de ocupación por recinto
  3. Resistencia al fuego de edificaciones
  4. Aplicación DS 43/2015 (bodegas) y DS 594 (habitabilidad)
8. Aclaraciones
Agradeceremos aclarar los siguientes puntos:
  1. Coordinación Proceso - Edificaciones : Coherencia entre disciplinas
    1. Confirmar que layout 4350 es versión vigente del proyecto
    2. Validar coherencia entre P&ID (4340) y layout general
    3. Indicar si modelo 3D arquitectura (5210) está actualizado
    4. Confirmar consistencia entre memorias y planos
  2. Validar ingeniería que se presentará para IFC : Validación de la ingeniería que se presentará en en el IFC. En el entendido que el IFC actua como permiso habilitante de las construcciones en área rural y diferencias respecto de lo que se presentará en el permiso generan riegos
    1. Indicar qué documentos corresponden a base oficial para IFC
    2. Confirmar nivel de ingeniería actual (IB vs IFC)
    3. Definir brechas identificadas por el cliente
    4. Confirmar estrategia de desarrollo del expediente DOM
